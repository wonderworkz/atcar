# 차살때 신규 랜딩페이지 리뉴얼 프로젝트 기획서

## 프로젝트 개요

### 목표
기존 차살때(https://chasalddae.com/) 랜딩페이지의 콘텐츠를 유지하면서, 동일한 레퍼런스 사이트의 디자인을 활용하여 반응형 랜딩페이지를 새롭게 구축

### 기술 스택
- **프론트엔드**: React + Tailwind CSS
- **애니메이션**: Framer Motion (스크롤 애니메이션, 호버 효과)
- **백엔드 연동**: 기존 차살때 시스템 API 활용
- **배포**: Vercel 또는 기존 인프라

### 5-2. 기존 시스템 데이터 연동 분석

**확인 필요 사항:**
1. **기존 API 엔드포인트 확인**
   - 문의 폼 제출 API URL
   - 필수 요청 파라미터
   - 인증 방식 (API Key, Token 등)

2. **데이터 암호화 방식**
   - 현재 사용 중인 암호화 알고리즘 (AES, RSA 등)
   - 암호화 키 관리 방식
   - 전송 시 HTTPS 사용 여부

3. **데이터 전송 프로세스**
   ```
   예상 플로우:
   사용자 입력 → 클라이언트 측 검증 → 민감정보 암호화
   → API 전송 (HTTPS) → 백엔드 복호화 → 데이터베이스 저장
   → CRM 시스템 또는 이메일 알림
   ```

4. **보안 조치**
   - CSRF 토큰 사용 여부
   - Rate limiting 설정
   - 개인정보 보호법 준수 (동의 체크박스, 개인정보처리방침)

**구현 계획:**
```javascript
// utils/api.js
import CryptoJS from 'crypto-js';

export const submitConsultation = async (formData) => {
  // 1. 민감정보 암호화
  const encryptedData = encryptPersonalInfo(formData);

  // 2. API 호출
  const response = await fetch('기존_API_엔드포인트', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer API_KEY',
    },
    body: JSON.stringify(encryptedData),
  });

  return response.json();
};

// utils/encryption.js
export const encryptPersonalInfo = (data) => {
  const secretKey = process.env.REACT_APP_ENCRYPTION_KEY;
  return {
    ...data,
    phone: CryptoJS.AES.encrypt(data.phone, secretKey).toString(),
    name: CryptoJS.AES.encrypt(data.name, secretKey).toString(),
  };
};
```

**기존 시스템 연동을 위한 필요 정보:**
- API 엔드포인트 URL
- API 인증 키/토큰
- 요청/응답 데이터 스키마
- 암호화 키 및 알고리즘
- 에러 핸들링 규칙
